# Coolify Service Credentials

Patterns for finding and recovering credentials for services deployed through Coolify (Umami, Plausible, NocoDB, etc.).

## How Coolify Stores Service Credentials

When Coolify deploys a service template, it auto-generates credentials and injects them as container environment variables. These are visible via `docker inspect`.

### Common env var patterns

| Pattern | Example | Source |
|---------|---------|--------|
| `SERVICE_PASSWORD_<NAME>` | `SERVICE_PASSWORD_POSTGRES=c05rvUcVRJRRmG3pFgQzxfw9zS1lpIPu` | Auto-generated by Coolify |
| `SERVICE_USER_<NAME>` | `SERVICE_USER_POSTGRES=aPHrQ9kDQUszGikT` | Auto-generated |
| `SERVICE_URL_<NAME>` | `SERVICE_URL_UMAMI=http://umami-xxx.ssl.ip.io` | Auto-generated |
| `SERVICE_FQDN_<NAME>` | `SERVICE_FQDN_UMAMI=umami.vivalto.no` | From user-set domain |

### How to inspect

```bash
# Find the container name
docker ps --format "{{.Names}}" | grep -i <service>

# Dump all env vars
docker inspect <container> --format '{{json .Config.Env}}' | python3 -c "
import json, sys
envs = json.load(sys.stdin)
for e in sorted(envs):
    print(e)
"
```

## Umami v3 Credentials

### Default credentials

Umami v3 uses **username** (NOT email) for login:
- **Username:** `admin`
- **Password:** `umami`

This differs from Umami v2 which used `admin@umami.is`.

When deployed via Coolify service template, the admin user may be seeded with `umami` as the password. If it doesn't work, it's faster to reset it via the database than to guess.

### Resetting the admin password

Umami stores user credentials in a PostgreSQL database managed by Coolify.

**Step 1: Find the PostgreSQL container and credentials**

```bash
# Postgres container is usually named <service>-<uuid>
docker ps --format "{{.Names}}" | grep postgres

# Get DB credentials from its env
docker inspect <postgres-container> --format '{{json .Config.Env}}' | python3 -c "
import json, sys
envs = json.load(sys.stdin)
for e in envs:
    if 'POSTGRES' in e or 'SERVICE_USER' in e or 'SERVICE_PASSWORD' in e:
        print(e)
"
```

**Step 2: Generate a bcrypt password hash**

On the Hermes host (or anywhere with Python + bcrypt):

```python
import bcrypt
password = b"<new-password>"
salt = bcrypt.gensalt(rounds=10)
hashed = bcrypt.hashpw(password, salt)
print(hashed.decode())
```

Install bcrypt if needed:
```bash
pip3 install bcrypt
```

**Step 3: Update the password in PostgreSQL**

```bash
docker exec <postgres-container> psql -U <user> -d umami -c \
  "UPDATE \"user\" SET password = '<bcrypt-hash>', updated_at = NOW() WHERE username = 'admin';"
```

Verify:
```bash
docker exec <postgres-container> psql -U <user> -d umami -c \
  "SELECT username, role, updated_at FROM \"user\";"
```

**Step 4:** Log in at `https://<domain>/login` with `admin` / `<new-password>`.

### Umami DB schema (users table)

```
Table: user
Columns: user_id (uuid PK), username, password (bcrypt), role, created_at, updated_at, deleted_at, display_name, logo_url
Indexes: user_pkey, user_user_id_key, user_username_key (unique)
```

## General Service Credential Recovery

For any Coolify service template:

1. **Check container env vars first** — `SERVICE_PASSWORD_*`, `SERVICE_USER_*`, `SERVICE_URL_*`
2. **Check Coolify UI** — App → Configuration → scroll to see auto-generated fields (may show PostgreSQL passwords, etc.)
3. **Check Coolify UI → Environment Variables** tab — lists all injected env vars
4. **Direct DB access** — Coolify services typically run PostgreSQL alongside. Grab the `POSTGRES_USER` and `POSTGRES_PASSWORD` from the DB container, then query the application database directly.
